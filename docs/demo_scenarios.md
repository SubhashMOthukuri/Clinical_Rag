# MedReconcile AI — Interview Demo Scenarios

Five scenarios that cover the full surface area of the pipeline:
happy path → polypharmacy → safe pair → invalid input → unknown drug.

---

## Scenario A — Critical Two-Drug Interaction (Bleeding Risk)

**Clinical story:** A nurse enters a patient on warfarin who was just given ibuprofen for pain.
The system catches a dangerous anticoagulant + NSAID combination in under 10 seconds.

**Input**
```json
POST /reconcile
{
  "medications": [
    { "name": "warfarin",  "dose": 5.0,   "unit": "mg" },
    { "name": "ibuprofen", "dose": 400.0, "unit": "mg" }
  ]
}
```

**Output**
```json
{
  "status": "SUCCESS",
  "total_medications": 2,
  "total_warnings": 1,
  "critical_warnings": 1,
  "warnings": [
    {
      "drugs_involved": ["warfarin", "ibuprofen"],
      "severity": "RED",
      "action": "MONITOR",
      "reaction_result": "The concomitant use of warfarin and ibuprofen may increase the risk of bleeding due to the additive effects of these medications. Ibuprofen, a non-steroidal anti-inflammatory agent, can increase the risk of bleeding when used with warfarin, an anticoagulant.",
      "nurse_summary_to_doctor": "Patient is taking warfarin and ibuprofen, which may increase the risk of bleeding. Close monitoring of INR and patient's condition is recommended.",
      "citation": ["article-31296_chunk_0037", "article-31294_chunk_0001", "article-31296_chunk_0002"],
      "confidence": 0.9,
      "data_source": "FRESH_FDA"
    }
  ],
  "response_time_ms": 8089
}
```

**What to highlight in the interview**
- `severity: RED` + `action: MONITOR` generated by the LLM from real StatPearls clinical literature
- `citation` entries are traceable chunk IDs — the nurse can see exactly which article was used
- `nurse_summary_to_doctor` is a plain-English escalation note, not a raw drug label
- `confidence: 0.9` reflects RAG retrieval quality, not a hardcoded value

## Scenario B — Life-Threatening Serotonin Syndrome Risk

**Clinical story:** Patient on sertraline (SSRI) is prescribed tramadol (opioid) for post-op pain.
The system flags a potential life-threatening interaction with `action: STOP`.

**Input**
```json
POST /reconcile
{
  "medications": [
    { "name": "sertraline", "dose": 50.0, "unit": "mg" },
    { "name": "tramadol",   "dose": 50.0, "unit": "mg" }
  ]
}
```

**Output**
```json
{
  "status": "SUCCESS",
  "total_warnings": 1,
  "critical_warnings": 1,
  "warnings": [
    {
      "drugs_involved": ["sertraline", "tramadol"],
      "severity": "RED",
      "action": "STOP",
      "reaction_result": "The concomitant use of sertraline and tramadol poses a significant risk for serotonin syndrome, a potentially life-threatening condition. Symptoms may include mental status changes, autonomic instability, neuromuscular symptoms, seizures, and/or gastrointestinal symptoms.",
      "nurse_summary_to_doctor": "Patient is at risk for serotonin syndrome due to concurrent use of sertraline and tramadol. Immediate discontinuation of one of the medications and close monitoring is recommended.",
      "citation": ["article-173600_chunk_0052", "article-19133_chunk_0055"],
      "confidence": 0.95,
      "data_source": "FRESH_FDA"
    }
  ],
  "response_time_ms": 2398
}
```

**What to highlight**
- `action: STOP` vs `MONITOR` — the LLM distinguishes severity of required response
- `confidence: 0.95` — highest of all scenarios, driven by strong RAG evidence in StatPearls
- 2.4 s end-to-end: RxNorm → FDA → Pinecone → LLM → structured response

## Scenario C — Polypharmacy (3 Drugs, 3 Pairs Checked in Parallel)

**Clinical story:** A complex patient arrives on warfarin, aspirin, and ibuprofen.
The system checks all C(3,2) = 3 pairs concurrently and returns all warnings in one call.

**Input**
```json
POST /reconcile
{
  "medications": [
    { "name": "warfarin",  "dose": 5.0,   "unit": "mg" },
    { "name": "aspirin",   "dose": 81.0,  "unit": "mg" },
    { "name": "ibuprofen", "dose": 400.0, "unit": "mg" }
  ]
}
```

**Output**
```json
{
  "status": "SUCCESS",
  "total_medications": 3,
  "total_warnings": 3,
  "critical_warnings": 3,
  "warnings": [
    {
      "drugs_involved": ["warfarin", "aspirin"],
      "severity": "RED",
      "action": "MONITOR",
      "reaction_result": "Concurrent use of warfarin and aspirin increases bleeding risk via synergistic effects on hemostasis and reduced clotting factor synthesis.",
      "citation": ["article-31296_chunk_0037", "article-31296_chunk_0002", "article-31294_chunk_0001"],
      "confidence": 0.9
    },
    {
      "drugs_involved": ["warfarin", "ibuprofen"],
      "severity": "RED",
      "action": "MONITOR",
      "reaction_result": "Warfarin and ibuprofen together increase bleeding risk via additive anticoagulant and NSAID effects.",
      "citation": ["article-31296_chunk_0037", "article-31294_chunk_0001", "article-31296_chunk_0002"],
      "confidence": 0.9
    },
    {
      "drugs_involved": ["aspirin", "ibuprofen"],
      "severity": "RED",
      "action": "STOP",
      "reaction_result": "Concurrent aspirin and ibuprofen may cause severe GI bleeding. Risk is elevated in patients ≥60, with history of ulcers, or on other blood-thinning agents. Severe toxicity may require IV sodium bicarbonate or hemodialysis.",
      "citation": ["article-30366_chunk_0053", "article-470_chunk_0031"],
      "confidence": 0.9
    }
  ],
  "response_time_ms": 2449
}
```

**What to highlight**
- 3 LLM calls ran **in parallel** — 2.4 s total, not 3× serial latency
- Each pair gets independent citations from separate StatPearls articles
- `aspirin + ibuprofen` escalates to `STOP` with a clinical management note (hemodialysis)
- This is the core value proposition: a nurse submits one list, gets a complete picture

## Scenario D — Safe Combination (No False Positives)

**Clinical story:** Standard diabetic hypertension regimen — metformin + lisinopril.
The system confirms no interaction, returning cleanly with zero warnings.

**Input**
```json
POST /reconcile
{
  "medications": [
    { "name": "metformin",  "dose": 500.0, "unit": "mg" },
    { "name": "lisinopril", "dose": 10.0,  "unit": "mg" }
  ]
}
```

**Output**
```json
{
  "status": "SUCCESS",
  "total_medications": 2,
  "total_warnings": 0,
  "critical_warnings": 0,
  "warnings": [],
  "unverified_drugs": [],
  "response_time_ms": 355
}
```

**What to highlight**
- 355 ms — no interaction evidence found, pipeline short-circuits before LLM call
- Empty `warnings: []` is the correct clinical answer, not a failure
- Demonstrates the system does **not** over-flag — precision matters as much as recall
- Both drugs resolve to valid RxCUIs via RxNorm (`metformin: 6809`, `lisinopril: 29046`)

## Scenario E — Invalid Input (Fails Closed, 422)

**Clinical story:** A bad API call is made — negative dose and an unrecognised unit.
The system rejects the request at Stage 1 before any external API is called.

**Input**
```json
POST /reconcile
{
  "medications": [
    { "name": "wa", "dose": -5.0, "unit": "tablespoon" }
  ]
}
```

**Output — HTTP 422**
```json
{
  "detail": [
    {
      "type": "greater_than",
      "loc": ["body", "medications", 0, "dose"],
      "msg": "Input should be greater than 0",
      "input": -5.0
    },
    {
      "type": "enum",
      "loc": ["body", "medications", 0, "unit"],
      "msg": "Input should be 'mg', 'mcg', 'g', 'mL', 'IU' or 'unit'",
      "input": "tablespoon"
    }
  ]
}
```

**What to highlight**
- All validation errors are returned **together** in one response, not one at a time
- `name: "wa"` would also fail (min length 3) — the full error list surfaces at once
- Nothing downstream (RxNorm, FDA, Pinecone, LLM) is called — zero cost, zero latency
- This is the **fail-closed** design: bad input never reaches the clinical pipeline
